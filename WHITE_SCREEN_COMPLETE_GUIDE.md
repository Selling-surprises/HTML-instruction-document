## 白屏问题的常见原因

### 1. 模块加载失败 ✅ 已修复
**症状：** 控制台显示 504 或模块加载错误  
**原因：** 某些 npm 包不适合在浏览器环境中使用  
**解决：** 已移除 html-to-docx 依赖

### 2. 缺少必要的 Provider ✅ 已修复
**症状：** 应用加载但内容不显示  
**原因：** 缺少 ThemeProvider 或其他必要的 Context Provider  
**解决：** 已添加 ThemeProvider 和 Toaster

### 3. 环境变量缺失 ✅ 已处理
**症状：** Supabase 初始化失败  
**原因：** VITE_SUPABASE_URL 或 VITE_SUPABASE_ANON_KEY 未配置  
**解决：** 已添加默认占位值和警告提示

### 4. 路由配置问题
**症状：** 特定路由白屏  
**原因：** 路由组件加载失败或路径配置错误  
**检查：** 查看 src/routes.tsx 和 src/App.tsx

### 5. 浏览器缓存
**症状：** 修复后仍然白屏  
**原因：** 浏览器缓存了旧的有问题的代码  
**解决：** 清除缓存并强制刷新

## 完整的排查步骤

### 步骤 1：清除缓存
```
1. 按 Ctrl+Shift+Delete (Windows) 或 Cmd+Shift+Delete (Mac)
2. 选择"缓存的图片和文件"
3. 选择"全部时间"
4. 点击"清除数据"
5. 关闭所有浏览器窗口
6. 重新打开浏览器
```

### 步骤 2：强制刷新
```
- Windows: Ctrl + F5 或 Ctrl + Shift + R
- Mac: Cmd + Shift + R
- 或者使用无痕模式打开
```

### 步骤 3：检查控制台
```
1. 按 F12 打开开发者工具
2. 切换到 Console 标签
3. 刷新页面
4. 查看是否有红色错误信息
```

### 步骤 4：检查网络请求
```
1. 在开发者工具中切换到 Network 标签
2. 刷新页面
3. 查看是否有失败的请求（红色标记）
4. 特别注意 .js 和 .css 文件的加载状态
```

## 特定场景的解决方案

### 场景 1：首次访问白屏
**可能原因：**
- 依赖加载失败
- 网络连接问题
- 浏览器不兼容

**解决方案：**
1. 检查网络连接
2. 尝试其他浏览器
3. 查看控制台错误信息

### 场景 2：刷新后白屏
**可能原因：**
- 浏览器缓存问题
- Service Worker 缓存了旧版本

**解决方案：**
1. 清除浏览器缓存
2. 使用无痕模式测试
3. 检查是否有 Service Worker（Application 标签）

### 场景 3：特定操作后白屏
**可能原因：**
- JavaScript 运行时错误
- 状态管理问题
- 内存溢出

**解决方案：**
1. 查看控制台错误
2. 刷新页面重试
3. 记录重现步骤

### 场景 4：移动端白屏
**可能原因：**
- 移动浏览器兼容性
- 视口设置问题
- 触摸事件冲突

**解决方案：**
1. 在桌面浏览器的移动模式测试
2. 检查 viewport meta 标签
3. 查看移动端控制台（使用 USB 调试）

## 常见错误信息解释

### "Failed to fetch"
**含义：** 网络请求失败  
**原因：** 网络连接问题或 CORS 错误  
**解决：** 检查网络连接，确认 API 地址正确

### "Cannot read property of undefined"
**含义：** 尝试访问未定义对象的属性  
**原因：** 数据未加载或组件状态错误  
**解决：** 添加空值检查，使用可选链操作符

### "Maximum update depth exceeded"
**含义：** 组件陷入无限渲染循环  
**原因：** useEffect 依赖项配置错误  
**解决：** 检查 useEffect 的依赖数组

### "Hydration failed"
**含义：** 服务端渲染和客户端渲染不匹配  
**原因：** 本应用不使用 SSR，不应出现此错误  
**解决：** 如果出现，可能是第三方库问题
