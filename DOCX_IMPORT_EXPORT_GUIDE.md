# Word 文档导入导出功能使用指南

## 功能概述

智能文档编辑器现已支持完整的 Word 文档（.docx 格式）导入导出功能，让您可以轻松地在编辑器和 Microsoft Word 之间转换文档。

## 主要特性

### ✨ 导出功能
- **标准 .docx 格式**：生成与 Microsoft Word 兼容的文档格式
- **完整的样式支持**：保留文本格式、标题、表格、列表等样式
- **图片处理**：自动处理图片尺寸，确保在 Word 中正常显示
- **纸张设置**：支持 A4、A3、Letter、Legal 等多种纸张尺寸
- **页面方向**：支持竖向和横向两种页面方向
- **自定义页边距**：默认 2.54cm 页边距，确保打印效果
- **浏览器兼容**：纯前端实现，无需服务器支持

### 📥 导入功能
- **格式支持**：支持 .docx 和 .doc 格式
- **智能转换**：使用 mammoth 库将 Word 文档转换为 HTML
- **样式保留**：尽可能保留原文档的格式和样式
- **错误处理**：提供友好的错误提示和警告信息

## 使用方法

### 导出 Word 文档

1. **打开设置面板**
   - 点击右下角的设置按钮（齿轮图标）
   - 或在移动端点击菜单按钮

2. **配置导出选项**（可选）
   - 在"Word 导出设置"部分
   - 选择纸张大小：A4、A3、Letter、Legal
   - 选择页面方向：竖向或横向

3. **导出文档**
   - 在"导出"部分
   - 点击"导出 Word (.docx)"按钮
   - 文档将自动下载到您的设备

### 导入 Word 文档

1. **打开设置面板**
   - 点击右下角的设置按钮

2. **选择文件**
   - 在"导入"部分
   - 点击"导入 Word"按钮
   - 选择要导入的 .docx 或 .doc 文件

3. **等待处理**
   - 系统会显示"正在导入"提示
   - 导入完成后会显示成功消息
   - 文档内容将替换当前编辑器内容

## 支持的元素

### ✅ 完全支持
- **文本格式**：粗体、斜体、下划线、删除线
- **标题**：H1-H6 所有级别的标题
- **段落**：普通段落、对齐方式
- **列表**：有序列表、无序列表
- **表格**：包含边框、单元格样式
- **链接**：超链接和链接卡片
- **图片**：自动转换为 base64 嵌入

### ⚠️ 部分支持
- **代码块**：导出为带样式的预格式化文本
- **复杂表格**：合并单元格可能需要调整
- **自定义样式**：部分 CSS 样式可能无法完全保留

### ❌ 不支持
- **宏和脚本**：Word 宏不会被导入
- **嵌入对象**：如 Excel 表格、图表等
- **页眉页脚**：暂不支持页眉页脚
- **批注和修订**：不保留批注和修订记录

## 最佳实践

### 导出建议

1. **图片优化**
   - 导出前确保图片大小合理
   - 过大的图片会增加文件大小
   - 建议图片宽度不超过 800px

2. **表格处理**
   - 复杂表格建议简化后再导出
   - 确保表格边框样式清晰
   - 避免过宽的表格

3. **格式检查**
   - 导出前预览文档内容
   - 检查标题层级是否正确
   - 确认链接是否有效

### 导入建议

1. **文件准备**
   - 使用 .docx 格式（推荐）
   - 确保文件未损坏
   - 文件大小建议不超过 10MB

2. **样式简化**
   - 复杂样式可能无法完全保留
   - 建议使用标准的 Word 样式
   - 避免使用过多自定义格式

3. **内容检查**
   - 导入后检查内容完整性
   - 必要时手动调整格式
   - 保存前预览最终效果

## 技术细节

### 导出实现
```typescript
// 使用 Word 兼容的 HTML 格式
// 通过特定的 XML 命名空间和样式，生成可被 Word 识别的文档

const htmlContent = `
  <html xmlns:o='urn:schemas-microsoft-com:office:office' 
        xmlns:w='urn:schemas-microsoft-com:office:word'>
  <head>
    <meta name="ProgId" content="Word.Document">
    <style>
      @page Section1 {
        size: 21.0cm 29.7cm;
        margin: 2.54cm;
      }
      /* 完整的样式定义 */
    </style>
  </head>
  <body>
    <div class="Section1">
      ${content}
    </div>
  </body>
  </html>
`;

// 创建 Blob 并下载
const blob = new Blob(['\ufeff', htmlContent], {
  type: 'application/vnd.openxmlformats-officedocument.wordprocessingml.document'
});
saveAs(blob, 'document.docx');
```

## 常见问题

### Q: 导出的文档在 Word 中打开样式不对？
A: 导出使用的是 Word 兼容的 HTML 格式。大部分样式会正确显示，但某些复杂的 CSS 样式可能需要在 Word 中进行微调。这是 HTML 到 Word 转换的正常现象。

### Q: 导入的文档丢失了部分格式？
A: mammoth 库会尽力保留格式，但某些复杂样式可能无法完全转换。您可以在导入后手动调整格式。

### Q: 图片在导出的文档中显示不正常？
A: 确保图片是有效的 URL。如果图片来自外部链接，Word 打开时需要网络连接才能显示。建议使用 base64 格式的图片以确保离线也能查看。

### Q: 导出的文件很大？
A: 如果文档包含大量图片，文件会比较大。建议：
- 压缩图片后再插入
- 使用适当的图片尺寸
- 考虑使用图片链接而非嵌入

### Q: 支持批量导入导出吗？
A: 目前仅支持单个文件的导入导出。如需批量处理，请分别操作每个文件。

### Q: 为什么不使用真正的 Office Open XML 格式？
A: 为了确保浏览器兼容性和性能，我们使用 Word 兼容的 HTML 格式。这种格式可以被所有主流 Office 软件正确打开和编辑，同时避免了复杂的依赖和构建问题。
